# Malyarka Clean: правила безопасного Excel/Corel export

Дата: 2026-06-14

Серия:

```text
BATCH_SERIES_048_050_SAFE_EXCEL_COREL_EXPORT_PLANNING
```

## Назначение

Этот документ фиксирует правила будущего Excel/Corel export до реализации.

Сейчас export не реализуется. Excel-файлы и Corel-файлы не создаются.

## Главное правило

Excel/Corel export разрешён только для чистого заказа:

```text
status = clean
disputed_rows пустой
export_blocked = false
```

Если заказ не чистый, export должен быть заблокирован.

## Когда export заблокирован

Export нельзя готовить, если:

- `status = has_disputes`;
- `status = empty_or_invalid`;
- есть `disputed_rows`;
- `export_blocked = true`.

## Что попадает в будущий export

В будущий Excel/Corel export могут попасть только подтверждённые строки.

Спорные строки не попадают в будущий Excel.

## Что нельзя менять ради export

Будущий export не должен менять:

- парсер;
- расчёт площади;
- правила спорных строк;
- статусы заказа;
- внутреннюю логику определения `clean`, `has_disputes`, `empty_or_invalid`.

Export должен быть отдельным слоем поверх уже готового результата заказа.

## Источник данных

Будущий export должен использовать уже готовые внутренние `corel_rows`.

Это значит:

- не разбирать заказ заново;
- не додумывать спорные строки;
- не исправлять пользовательский ввод внутри export;
- не считать стоимость;
- не считать материалы.

## Что такое будущий Excel

Будущий Excel-файл — это только технический файл для Corel.

Он не является:

- производственным расчётом стоимости;
- базой заказов;
- документом для клиента;
- расчётом ЛКМ;
- расчётом материалов.

## Что не входит

В эту плановую серию не входит:

- создание Excel-файла;
- создание Corel-файла;
- export на диск;
- Telegram;
- Vision;
- API;
- база данных;
- цены;
- стоимость;
- ЛКМ;
- материалы;
- производство;
- старая Малярка как активная система.

## Перед реализацией

Перед реализацией Excel/Corel export нужен отдельный технический пакет и отдельное разрешение пользователя.
