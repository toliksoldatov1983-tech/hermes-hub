# Шаблон первой папки заказа v0.1

Дата: 2026-06-14

Серия:

```text
Серия 090-092 - План шаблона первой папки заказа
BATCH_SERIES_090_092_ORDER_FOLDER_TEMPLATE_PLANNING
```

## Важная граница

Это только текстовый план.

В этой серии не создаются реальные папки заказа внутри `E:\РАБОТА`.
Не создаются папки годов и месяцев.
Старые заказы не переносятся.
`E:\Заказы 2026` не трогается.

## Будущий путь нового заказа

Будущий новый заказ может располагаться так:

```text
E:\РАБОТА\01_ЗАКАЗЫ\2026\06_Июнь\Название_заказа
```

Этот путь пока является только примером шаблона.
Папка `Название_заказа`, папка `2026` и папка `06_Июнь` сейчас не создаются.

## Структура одной папки заказа

Шаблон папки одного заказа v0.1:

```text
Название_заказа
├─ 01_Исходные
├─ 02_Corel
├─ 03_ArtCAM
├─ 04_Excel
└─ 05_Покраска
```

## Назначение папок

### 01_Исходные

`01_Исходные` - всё, что пришло от клиента:

- размеры;
- техническое задание;
- картинки;
- переписка;
- таблицы, если они есть отдельными файлами.

### 02_Corel

`02_Corel` - файлы и работа в Corel:

- `.cdr` файлы;
- отрисовка;
- раскладка;
- визуальные пометки;
- примечания внутри Corel.

### 03_ArtCAM

`03_ArtCAM` - подготовка под ЧПУ:

- `.art` файлы;
- подготовка под CNC / ЧПУ;
- отдельные операции ArtCAM, если они нужны.

### 04_Excel

`04_Excel` - таблицы заказа:

- Excel-файлы заказа;
- списки деталей;
- будущий экспорт в Corel;
- рабочие таблицы.

### 05_Покраска

`05_Покраска` - данные для малярного цеха, если заказ связан с покраской или цветом:

- цвет;
- номер краски;
- материал;
- толщина;
- данные для покраски.

## Решения v0.1

- Папка `Фото_заметки` пока не входит в обязательный шаблон.
- Визуальные заметки можно хранить внутри Corel-файла.
- Сложность заказа фиксируется в карточке заказа, а не через новые верхние папки.
- Итоговый файл заказа пока не создаётся, но остаётся будущим этапом.
- Шаблон одной папки заказа принимается как текстовый план v0.1.

## Будущая карточка заказа

В будущем внутри папки заказа может появиться карточка заказа.
Сейчас она не создаётся как реальный файл в `E:\РАБОТА`.

Возможная будущая идея:

```text
Название_заказа\ORDER_CARD.md
```

Будущая карточка заказа должна фиксировать:

- название заказа;
- клиент;
- дата;
- тип изделия;
- материал;
- толщина;
- цвет;
- номер краски;
- тип обработки;
- Corel-файл;
- ArtCAM-файл;
- Excel-файл;
- покраска / не покраска;
- статус заказа;
- примечания;
- позже: цена, зарплаты, склад, материалы, итоговая экономика заказа.

## Что не реализуется сейчас

Сейчас не создаются:

- папка `Название_заказа`;
- папка `Абай_планки`;
- папка `Петр_столики`;
- папки годов внутри `E:\РАБОТА`;
- папки месяцев внутри `E:\РАБОТА`;
- реальная карточка заказа;
- итоговый файл заказа.

Сейчас не выполняются:

- перенос старых заказов;
- переименование файлов;
- изменение `E:\Заказы 2026`;
- действия с Corel / ArtCAM / CNC;
- Excel/Corel automation;
- расчёты цен, зарплат, склада, материалов или ЛКМ.

## Итог

Шаблон папки заказа v0.1 зафиксирован как документация.
Следующий шаг должен быть отдельным решением пользователя перед созданием реальной первой папки заказа.
