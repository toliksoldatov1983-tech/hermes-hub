# OBSIDIAN LONG MEMORY RULES — Правила для долговременной памяти

Дата: 2026-06-21
Применение: Obsidian vault `E:\Hermes-General\obsidian-long-memory\`

---

## Назначение

Long memory = **устойчивые знания**, которые не устаревают каждый день:
- Архитектура проекта.
- Бизнес-правила и стандарты.
- Технические знания (материалы, технологии).
- Принятые решения (итоговые, не черновики).
- Шаблоны документов.

---

## Главное правило

```
Hermes Hub — source of truth (оперативный state).
Obsidian — объясняющая память (knowledge, not state).
```

---

## 🟢 Что хранить в Long Memory

| Папка | Что | Пример |
|-------|-----|--------|
| `00_Profile` | О себе, роли, контакты | Имя, Telegram, сервер, партнёры |
| `01_Rules` | Бизнес-правила и стандарты | Как считать площадь, нормы расхода, цены |
| `02_Knowledge` | Технические знания | Типы пластика, технология покраски, фрезеровка |
| `03_Orders` | **Шаблоны и типовые примеры.** НЕ реальные заказы. | Типовой заказ: 700×500 |
| `04_Clients` | Информация о клиентах | Постоянные клиенты, предпочтения |
| `05_Projects` | Описание проектов, архитектура | Hermes Hub, Malyarka Clean, структура |
| `06_Processes` | Бизнес-процессы | Как принять заказ → произвести → отдать |
| `07_Analytics` | Аналитика и выводы | Средний чек, популярные размеры, тренды |
| `08_Templates` | Шаблоны документов | Договор, счёт, коммерческое предложение |
| `09_Environment` | Окружение и инструменты | Список софта, сервер, станки |
| `10_Sessions` | Итоги сессий / недель | Что сделано за неделю, выводы |

---

## 🚫 Что НЕ хранить в Long Memory

| Не хранить | Почему | Где должно быть |
|-----------|--------|-----------------|
| **Runtime-state** | Устаревает каждый час | `HERMES_HUB_STATE.md` |
| **Статус бота** | Меняется постоянно | `sync/SHARED_CURRENT_STATUS.md` |
| **Список задач** | Меняется каждый день | `TASK_QUEUE.md` |
| **Блокеры** | Меняются часто | `sync/BLOCKERS.md` |
| **Approval gates** | Должны быть в Hub | `sync/APPROVAL_GATES.md` |
| **Worklog** | Дублирование | `logs/WORKLOG.md` |
| **Секреты** | `.env`, token, ключи | **Нигде в Obsidian** |
| **Базы данных** | `orders.db` | **Нигде в Obsidian** |
| **Логи** | Live logs | **Нигде в Obsidian** |
| **Реальные заказы** | Клиентские данные | **Нигде в Obsidian** |

## 🔄 Как переносить решения из Hub в Long Memory

1. Решение принято → записано в `decisions/DECISIONS.md` (Hub).
2. Батч завершён → решение подтверждено практикой.
3. Раз в неделю: перенести **итоговые** решения в Long Memory.
4. В Long Memory — только **вывод**, не процесс обсуждения.

### Пример

```
Hub (DECISIONS.md):
2026-06-21 — Решение: использовать openpyxl для Файла Малярки.

Obsidian (01_Rules):
Файл Малярки формируется через openpyxl 3.1.5.
Заголовки на русском. Статусы: «чистый», «спорный», «пустой».
```

## 🔍 Еженедельная сверка

1. Открыть `HERMES_HUB_STATE.md`.
2. Открыть Long Memory.
3. Найти факты, которые есть в обоих местах.
4. Если факт в Hub актуальнее → обновить Long Memory.
5. Если факт только в Long Memory → перенести в Hub.

**Правило:** Hub всегда актуальнее. Long Memory — производная.
