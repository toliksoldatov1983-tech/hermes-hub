# Malyarka Clean: текущий рабочий путь пользователя

Дата: 2026-06-14

Серия:

```text
BATCH_SERIES_054_056_EXCEL_COREL_EXPORT_ACCEPTANCE_AND_NEXT_LAYER
```

## Назначение

Этот документ описывает, как пользователь сейчас работает с локальной Malyarka Clean.

## Текущий путь

1. Открыть входной файл заказа:

```text
notepad E:\Hermes-Hub\inputs\ORDER_INPUT.txt
```

2. Вставить текст заказа.

3. Сохранить файл.

4. Запустить проверку заказа:

```text
E:\Hermes-Hub\ПРОВЕРИТЬ_ЗАКАЗ_ИЗ_ФАЙЛА.cmd
```

5. Открыть последний текстовый результат:

```text
notepad E:\Hermes-Hub\outputs\LAST_ORDER_RESULT.txt
```

6. Посмотреть `status`.

Если:

```text
status = clean
```

можно запустить создание Excel для Corel:

```text
E:\Hermes-Hub\СОЗДАТЬ_EXCEL_ДЛЯ_COREL.cmd
```

7. Забрать Excel-файл:

```text
E:\Hermes-Hub\outputs\COREL_EXPORT.xlsx
```

8. Если есть спорные строки, Excel не создаётся.

В этом случае нужно:

1. Исправить `ORDER_INPUT.txt`.
2. Сохранить файл.
3. Повторить проверку.
4. Создать Excel только после `status = clean`.

## Что сейчас уже удобно

- Заказ можно вставить в обычный `.txt`.
- Результат сохраняется в обычный `.txt`.
- Excel создаётся только когда заказ чистый.
- Спорные строки блокируют Excel.

## Что пока неудобно

- Нужно помнить несколько запускателей.
- Проверка заказа и создание Excel — два отдельных действия.
- Пользователь должен сам понять, когда запускать Excel.

## Вывод

Следующий удобный слой должен объединить проверку и создание Excel в один запуск.
