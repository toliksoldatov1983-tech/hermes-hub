# Malyarka Clean: варианты следующего слоя после Excel export

Дата: 2026-06-14

Серия:

```text
BATCH_SERIES_054_056_EXCEL_COREL_EXPORT_ACCEPTANCE_AND_NEXT_LAYER
```

## Назначение

Этот документ описывает варианты следующего удобного слоя после локальной v0.1 и безопасного Excel/Corel export.

## 1. Один главный запускатель

Идея:

```text
проверить заказ и создать Excel если можно
```

Что должен делать будущий запускатель:

- читать `ORDER_INPUT.txt`;
- проверить заказ;
- сохранить `LAST_ORDER_RESULT.txt`;
- если `status = clean` — создать `COREL_EXPORT.xlsx`;
- если есть спорные строки — не создавать Excel и показать причину.

Плюс:

- меньше ручных действий;
- пользователю проще;
- меньше риска запустить не тот файл.

## 2. Улучшение локального меню

Можно позже сделать простое локальное меню:

- открыть файл заказа;
- проверить заказ;
- создать Excel;
- открыть результат;
- открыть инструкцию.

Плюс:

- меньше путаницы;
- все действия в одном месте.

## 3. Архивирование результатов заказов

Сейчас `LAST_ORDER_RESULT.txt` перезаписывается.

Можно позже сохранять историю:

- дата;
- исходный заказ;
- результат;
- статус;
- путь к Excel, если был создан.

Плюс:

- результаты не теряются;
- можно вернуться к старым заказам.

## 4. Подготовка к Telegram

Telegram можно рассматривать позже.

Пока нельзя:

- запускать бота;
- polling;
- API;
- реальные заказы через Telegram.

Сначала нужен отдельный план.

## 5. Подготовка к Vision

Vision можно рассматривать позже для изображений, скринов или фото заказов.

Пока нельзя:

- подключать Vision;
- отправлять изображения во внешние API;
- делать автоматическое распознавание.

Сначала нужен отдельный план.

## 6. База заказов

База может понадобиться для истории заказов.

Риски:

- данные клиентов;
- миграции;
- потеря данных;
- связь со старой Маляркой.

Сначала нужен отдельный план.

## 7. Цены и расчёты

Цены, стоимость, ЛКМ и материалы — отдельный рискованный слой.

Пока не делать:

- расчёт стоимости;
- прайсы;
- ЛКМ;
- материалы;
- производственные решения.

## Рекомендованный следующий этап

Рекомендуется:

```text
Серия 057–060 — Единый локальный запуск заказа
BATCH_SERIES_057_060_SINGLE_LOCAL_ORDER_RUNNER
```

Смысл будущего этапа:

- один запускатель читает `ORDER_INPUT.txt`;
- проверяет заказ;
- сохраняет `LAST_ORDER_RESULT.txt`;
- если заказ чистый — создаёт `COREL_EXPORT.xlsx`;
- если заказ спорный или пустой — не создаёт Excel и показывает причину.

## Важно

В серии 054–056 это только план.

Ничего не реализуется.
