# Malyarka Clean: приёмка сохранения результата и план входного файла заказа

Дата: 2026-06-13

Пакет:

```text
BATCH_035_SAVE_RESULT_ACCEPTANCE_AND_INPUT_FILE_PLANNING
```

## 1. Что принято

Сохранение результата заказа в текстовый файл считается принятым.

Работает:

- результат не теряется после закрытия окна;
- `LAST_ORDER_RESULT.txt` можно открыть через Блокнот;
- три базовых сценария сохраняются корректно.

Файл последнего результата:

```text
E:\Hermes-Hub\outputs\LAST_ORDER_RESULT.txt
```

Открыть через Win+R:

```text
notepad E:\Hermes-Hub\outputs\LAST_ORDER_RESULT.txt
```

## 2. Зачем нужен входной файл заказа

Входной файл заказа нужен, чтобы:

- не вставлять заказ в консоль каждый раз;
- можно было заранее подготовить заказ в Блокноте;
- запуск стал проще: положил текст заказа в файл -> запустил проверку -> получил результат.

## 3. Планируемый входной файл

В будущем можно использовать файл:

```text
E:\Hermes-Hub\inputs\ORDER_INPUT.txt
```

В этом пакете папка `inputs` и файл `ORDER_INPUT.txt` не создаются.

## 4. Что должен делать будущий режим

Будущий режим должен:

- читать текст заказа из `ORDER_INPUT.txt`;
- обрабатывать его через уже существующее ядро;
- показывать результат на экране;
- сохранять результат в `LAST_ORDER_RESULT.txt`.

## 5. Правила

- Использовать обычный `.txt`.
- Не создавать Excel.
- Не создавать Corel-файл.
- Не трогать базу данных.
- Если папки `inputs` нет, её можно создать в следующем техническом пакете.
- Пример входного заказа можно будет хранить в `ORDER_INPUT.txt`.

## 6. Что не входит

- Telegram;
- Vision;
- API;
- база данных;
- Excel-файл;
- Corel-файл;
- экспорт на диск для производства;
- цены;
- стоимость;
- ЛКМ;
- материалы;
- производство;
- CNC-обучение;
- новые агенты.
