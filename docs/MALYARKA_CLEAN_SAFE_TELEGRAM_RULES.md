# Malyarka Clean — правила безопасного Telegram-слоя

Дата: 2026-06-14

Серия:

```text
Серия 067–069 — План безопасного Telegram-слоя
BATCH_SERIES_067_069_SAFE_TELEGRAM_LAYER_PLANNING
```

Пакет:

```text
Пакет 067 — правила безопасного Telegram-слоя
```

## Цель

Описать границы будущего Telegram-слоя так, чтобы он стал только удобной оболочкой над уже работающей локальной версией Malyarka Clean.

Текущая рабочая локальная цепочка:

```text
ORDER_INPUT.txt
-> ЗАПУСТИТЬ_ЗАКАЗ.cmd
-> LAST_ORDER_RESULT.txt
-> COREL_EXPORT.xlsx только если status = clean
```

Telegram-слой не должен менять смысл этой цепочки.

## Главный принцип

Telegram должен использовать существующее ядро Malyarka Clean.

Он не должен становиться новой отдельной логикой расчёта заказа.

Правильная роль Telegram:

```text
принять текст заказа
-> передать текст в существующее ядро
-> показать пользователю результат ядра
```

## Что Telegram не должен менять

Будущий Telegram-слой не должен менять:

- парсер размеров;
- расчёт площади;
- правила спорных строк;
- статусы `clean`, `has_disputes`, `empty_or_invalid`;
- правило блокировки export при спорных строках;
- правило создания Excel только для `clean`;
- формат будущих Corel-строк;
- текущий локальный запуск без отдельного плана.

## Excel и Corel export

Telegram не должен трогать Excel/Corel export без отдельного плана.

На первом Telegram-этапе достаточно сообщать:

```text
Заказ clean. Excel можно создать локальным запускателем.
```

Отправка файлов через Telegram, автоматическое создание Excel из Telegram и любые изменения `COREL_EXPORT.xlsx` должны быть отдельными будущими сериями.

## Секреты и токены

В рамках безопасного Telegram-слоя нельзя:

- читать `.env`;
- менять `.env`;
- показывать Telegram token в документах или отчётах;
- искать реальные токены без отдельного разрешения;
- вставлять токен в код;
- хранить токен в `.cmd`;
- трогать ключи, пароли или другие секреты.

## Live-запуск

Нельзя запускать:

```text
Telegram
polling
webhook
live bot
external API
```

Первый технический этап после этого плана должен быть только `check` или `dry-run` режимом без подключения к Telegram.

## Старый bot.py

Старый `bot.py` нельзя использовать как активную систему.

Он не должен быть:

- точкой запуска;
- основой нового Telegram-слоя;
- источником токена;
- источником скрытой логики.

Если когда-нибудь понадобится смотреть старый `bot.py`, это должен быть отдельный read-only аудит с отдельным разрешением пользователя.

## Запретные зоны

В рамках Telegram-планирования и первого безопасного каркаса нельзя трогать:

```text
.env
orders.db
.git
tokens
keys
old bot.py
old Malyarka as active system
Telegram polling
Vision
external API
database
prices
cost
LKM
materials
Docker
commits/push

parser

dispute rules
production export
```

## Разрешённая будущая архитектурная идея

Безопасная идея будущего слоя:

```text
new telegram adapter
-> existing Malyarka Clean core
-> formatted text response
```

Telegram должен быть адаптером ввода/вывода, а не новым ядром.

## Итог

Telegram можно планировать как удобный слой над Malyarka Clean, но только при строгом соблюдении границ:

- использовать существующее ядро;
- не менять правила обработки заказа;
- не читать секреты;
- не запускать live Telegram;
- не использовать старый `bot.py`;
- не трогать Excel/Corel export без отдельного плана.
