# HEALTH CHECK DESIGN — Проект безопасной проверки бота

Дата: 2026-06-21
Статус: PLAN-ONLY. Не реализовано. Требует Codex для серверной части.
Применение: `systemctl status` + Telegram-тест + проверка флагов.

---

## Принцип

Health check проверяет, жив ли бот, **не читая secrets, DB, логи и реальные заказы.**
Только presence-only и статусы.

---

## 🩺 Что проверять

### 1. Service Status (сервер)

| Проверка | Команда | Ожидаемый результат |
|----------|---------|-------------------|
| Сервис запущен | `systemctl is-active malyarka-telegram-bot` | `active` |
| PID существует | `systemctl show -p MainPID malyarka-telegram-bot` | Число > 0 |
| Процесс жив | `ps -p <PID>` | Процесс существует |
| Потребление памяти | `ps -o rss= -p <PID>` | В пределах нормы |

---

### 2. Autostart Status

| Проверка | Команда | Ожидаемый результат |
|----------|---------|-------------------|
| Autostart | `systemctl is-enabled malyarka-telegram-bot` | `disabled` (пока) |

---

### 3. Feature Flags

| Проверка | Что | Ожидаемый результат |
|----------|-----|-------------------|
| Hermes adapter | `_HERMES_ADAPTER_ENABLED` | `False` |
| Phase 2 | Статус | `OFF` |
| Production | Статус | `OFF` |

**Важно:** не читать `config.py` содержимое. Проверять через `systemctl show` или presence-only.

---

### 4. Доступность Telegram

| Проверка | Как | Ожидаемый результат |
|----------|-----|-------------------|
| Бот отвечает на `/start` | Пользователь пишет `/start` | Ответ в течение 5 секунд |
| Бот парсит заказы | Пользователь пишет `700 x 500` | Парсинг + кнопки |

**Важно:** Telegram-тест выполняет только пользователь. Hermes не отправляет сообщения в Telegram сам.

## 🚫 Что НЕ проверять (даже в health check)

| Не проверять | Причина |
|-------------|---------|
| Содержимое `.env` | Секреты |
| Содержимое `config.py` | Секреты |
| Токен бота | Секреты |
| `orders.db` | Клиентские данные |
| Live логи | Приватность |
| Реальные заказы | Клиентские данные |
| `malyarka_ai/` | Внешние API |
| `malyarka_vision/` | Внешние API |

## ⚡ Как будет работать (будущее)

1. Hermes получает команду `/health` от пользователя в Telegram.
2. Hermes делает read-only проверки (только `systemctl status`, presence-only).
3. Hermes просит пользователя: «Напишите /start боту для проверки доступности.»
4. Пользователь подтверждает: «Бот ответил.»
5. Hermes формирует отчёт.

---

## 🔧 Что нужно для реализации (Codex)

1. SSH read-only доступ к серверу (`РАЗРЕШАЮ_SSH_READ_ONLY`).
2. Скрипт `health_check.sh` на сервере: `systemctl status` + `is-enabled` + `is-active`.
3. Формат вывода: JSON или markdown.
4. Интеграция с Hermes: команда `/health` → запуск скрипта → отчёт.
