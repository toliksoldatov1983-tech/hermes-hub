# Malyarka Clean: приёмка читаемого результата и примеров заказов

Дата: 2026-06-13

Серия:

```text
BATCH_SERIES_041_044_READABLE_RESULT_AND_EXAMPLES
```

## Что принято

- Читаемый формат `LAST_ORDER_RESULT.txt` реализован.
- Примеры заказов созданы.
- Быстрые `.cmd`-проверки примеров созданы.
- Локальная v0.1 получила более удобный результат и набор примеров.

## Что создано

Примеры:

```text
E:\Hermes-Hub\examples\CLEAN_ORDER.txt
E:\Hermes-Hub\examples\DISPUTED_ORDER.txt
E:\Hermes-Hub\examples\EMPTY_OR_INVALID_ORDER.txt
E:\Hermes-Hub\examples\MIXED_SEPARATORS_ORDER.txt
```

Проверки:

```text
E:\Hermes-Hub\ПРОВЕРИТЬ_ПРИМЕР_ЧИСТЫЙ.cmd
E:\Hermes-Hub\ПРОВЕРИТЬ_ПРИМЕР_СПОРНЫЙ.cmd
E:\Hermes-Hub\ПРОВЕРИТЬ_ПРИМЕР_МУСОРНЫЙ.cmd
E:\Hermes-Hub\ПРОВЕРИТЬ_ПРИМЕР_РАЗНЫЕ_РАЗДЕЛИТЕЛИ.cmd
```

## Какие сценарии проверены

- Чистый заказ -> `clean`.
- Заказ с одной спорной строкой -> `has_disputes`.
- Полностью мусорный заказ -> `empty_or_invalid`.
- Разные поддерживаемые разделители -> `clean`.

## Что улучшено в результате

`LAST_ORDER_RESULT.txt` теперь содержит понятные блоки:

- короткий итог;
- исходный заказ;
- подтверждённые строки;
- спорные строки;
- Corel-строки;
- подсказку, что делать дальше.
