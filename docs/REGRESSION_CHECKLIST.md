# REGRESSION CHECKLIST — Проверки после изменений

Дата: 2026-06-21
Использовать: после каждого live-патча, restart, или включения feature flag.

---

## ⚡ Быстрая проверка (3 минуты)

| # | Проверка | Что делать | Ожидаемый результат |
|---|---------|-----------|-------------------|
| 1 | `/start` | Написать `/start` боту | Бот отвечает приветствием |
| 2 | `700 x 500` — вне заказа | Написать `700 x 500` без контекста заказа | Бот парсит и предлагает Excel/Файл |
| 3 | Новый заказ | Полный цикл: текст заказа → парсинг → кнопки | Все кнопки появляются |

---

## 📦 Полная проверка (10 минут)

| # | Проверка | Что делать | Ожидаемый результат |
|---|---------|-----------|-------------------|
| 1 | `/start` | Написать `/start` | Приветствие |
| 2 | `700 x 500` вне заказа | Написать размеры без заказа | Парсинг, кнопки Excel/Файл |
| 3 | Новый заказ | `Заказ: 700 x 500, 400 x 300` | Парсинг, список размеров |
| 4 | `700 x 500` в заказе | Добавить строку в заказ | Корректный парсинг |
| 5 | Excel для Corel | Нажать кнопку Excel | Скачивается .xlsx |
| 6 | Файл Малярки | Нажать кнопку «Скачать Файл Малярки» | Скачивается .xlsx |
| 7 | Русские заголовки | Открыть скачанный Файл Малярки | Заголовки на русском, не на английском |
| 8 | Статусы русские | Проверить столбец статусов в Файле | «чистый», «спорный», не «clean»/«disputed» |

---

## 🔒 Безопасность (каждый раз)

| # | Проверка | Как | Ожидаемый результат |
|---|---------|-----|-------------------|
| 1 | Feature flag OFF | `systemctl status` + логи | `_HERMES_ADAPTER_ENABLED = False` |
| 2 | Phase 2 OFF | Проверить, не включён ли adapter | Hermes-адаптер не обрабатывает сообщения |
| 3 | Production OFF | Проверить конфигурацию | Production режим выключен |
| 4 | Autostart disabled | `systemctl is-enabled malyarka-telegram-bot` | `disabled` |
| 5 | Service running | `systemctl is-active malyarka-telegram-bot` | `active` |

---

## 🛑 Если что-то не проходит

1. **Stop.** Не продолжать дальше.
2. Записать, какая проверка не прошла.
3. Сравнить с последним состоянием из `PATCH_LOG.md`.
4. Если патч вызвал проблему → rollback по `PATCH_LOG.md`.
5. Сообщить пользователю.
