# Malyarka Clean: варианты следующего крупного направления

Дата: 2026-06-13

Серия:

```text
BATCH_SERIES_045_047_LOCAL_V01_ACCEPTANCE_AND_ROADMAP
```

## 1. Excel/Corel export

Зачем нужен:

- чтобы переходить от проверки заказа к подготовке производственных данных.

Что даст пользователю:

- понятный файл или структуру для дальнейшей работы в Corel/Excel.

Почему рискованно:

- может создать производственные файлы;
- ошибка может попасть в реальный заказ.

Нельзя без отдельного разрешения:

- создавать Excel-файлы;
- создавать Corel-файлы;
- делать export на диск для производства.

## 2. Улучшение локального интерфейса

Зачем нужен:

- чтобы работать удобнее, без ручного запуска разных `.cmd`.

Что даст пользователю:

- более понятный локальный сценарий: открыть, вставить заказ, получить результат.

Почему рискованно:

- можно усложнить проект раньше времени;
- можно сломать простую рабочую v0.1.

Нельзя без отдельного разрешения:

- запускать приложение;
- менять архитектуру;
- переименовывать папки.

## 3. Telegram

Зачем нужен:

- чтобы принимать заказы и идеи с телефона.

Что даст пользователю:

- удалённый доступ к обработке заказов.

Почему рискованно:

- бот может отправлять данные наружу;
- нужны токены;
- можно случайно запустить старую Малярку.

Нельзя без отдельного разрешения:

- запускать Telegram-бота;
- трогать токены;
- использовать старый `bot.py`.

## 4. Vision

Зачем нужен:

- чтобы в будущем разбирать изображения, скрины или фото заказов.

Что даст пользователю:

- меньше ручного ввода, больше автоматизации.

Почему рискованно:

- изображения могут содержать личные данные;
- нужны внешние модели или API;
- результаты Vision могут ошибаться.

Нельзя без отдельного разрешения:

- подключать Vision;
- отправлять изображения наружу;
- использовать внешние API.

## 5. База заказов

Зачем нужна:

- чтобы хранить историю заказов, статусы и результаты.

Что даст пользователю:

- поиск, история, аналитика, долгосрочное накопление данных.

Почему рискованно:

- база может содержать важные данные клиентов;
- ошибка миграции может повредить историю.

Нельзя без отдельного разрешения:

- создавать или менять базу данных;
- трогать `orders.db`;
- переносить старые данные.

## 6. Цены и расчёты

Зачем нужны:

- чтобы считать стоимость заказа.

Что даст пользователю:

- быстрый предварительный расчёт.

Почему рискованно:

- ошибка цены напрямую влияет на деньги;
- правила могут быть сложными.

Нельзя без отдельного разрешения:

- считать стоимость;
- фиксировать прайсы;
- смешивать площадь и стоимость.

## 7. ЛКМ и материалы

Зачем нужны:

- чтобы учитывать расход материалов.

Что даст пользователю:

- планирование производства и закупок.

Почему рискованно:

- нормы расхода зависят от технологии;
- неправильные расчёты могут повлиять на производство.

Нельзя без отдельного разрешения:

- считать ЛКМ;
- учитывать материалы;
- запускать производственные сценарии.
