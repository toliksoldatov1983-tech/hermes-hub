# Malyarka Clean: план улучшения читаемости LAST_ORDER_RESULT.txt

Дата: 2026-06-13

Пакет:

```text
BATCH_040_RESULT_READABILITY_PLANNING
```

## Зачем улучшать читаемость

Сейчас `LAST_ORDER_RESULT.txt` содержит технические словари Python. Это полезно для проверки, но неудобно для обычного пользователя.

Нужно сделать результат более человеческим, чтобы было понятно:

- какой статус заказа;
- какие строки приняты;
- какие строки спорные;
- какая площадь;
- можно ли готовить Corel-строки;
- почему результат заблокирован.

## Что улучшить в будущем

Планируемый текстовый результат должен быть разделён на понятные блоки:

1. Короткий итог.
2. Исходный заказ.
3. Подтверждённые строки.
4. Спорные строки.
5. Площадь.
6. Corel-строки.
7. Что делать дальше.

## Пример будущего направления

Вместо технического вида:

```text
{'row_id': 'row-1', 'height': 1000, 'width': 400, 'quantity': 2}
```

Показывать:

```text
Строка 1: 1000 x 400, количество 2
```

## Что не входит

- Excel/Corel export;
- создание производственных файлов;
- Telegram;
- Vision;
- API;
- база данных;
- цены;
- стоимость;
- ЛКМ;
- материалы;
- CNC.
