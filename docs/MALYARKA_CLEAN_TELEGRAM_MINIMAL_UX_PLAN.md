# Malyarka Clean — минимальный UX Telegram

Дата: 2026-06-14

Серия:

```text
Серия 067–069 — План безопасного Telegram-слоя
BATCH_SERIES_067_069_SAFE_TELEGRAM_LAYER_PLANNING
```

Пакет:

```text
Пакет 068 — минимальный UX Telegram
```

## Цель

Описать самый простой пользовательский сценарий Telegram без файлов, фото, Vision, базы, цен и live-запуска.

Первый Telegram UX должен повторять смысл текущей локальной версии:

```text
пользователь даёт текст заказа
-> система проверяет заказ
-> система возвращает понятный результат
```

## Основной сценарий

1. Пользователь отправляет боту текст заказа одним сообщением.
2. Бот передаёт этот текст в существующее ядро Malyarka Clean.
3. Бот возвращает результат проверки.
4. Если заказ чистый, бот пишет, что Excel можно создать.
5. Если есть спорные строки, бот просит исправить только спорные строки.

## Что должен показывать бот

Минимальный ответ должен содержать:

- статус заказа;
- подтверждённые строки;
- спорные строки;
- общую площадь по подтверждённым строкам;
- заблокирован ли export;
- короткую подсказку, что делать дальше.

## Ответ для clean

Если `status = clean`, бот должен показать:

```text
Статус: clean
Подтверждённые строки: ...
Спорных строк нет.
Площадь: ... m2
Excel можно создать локальным запускателем.
```

На первом этапе бот не обязан сам создавать или отправлять Excel.

## Ответ для has_disputes

Если `status = has_disputes`, бот должен показать:

```text
Статус: has_disputes
Подтверждённые строки: ...
Спорные строки: ...
Площадь по подтверждённым строкам: ... m2
Excel заблокирован.
Исправь только спорные строки и отправь заказ снова.
```

Важно: бот не должен просить переписывать весь заказ, если достаточно исправить спорные строки.

## Ответ для empty_or_invalid

Если `status = empty_or_invalid`, бот должен показать:

```text
Статус: empty_or_invalid
Понятных строк не найдено.
Excel заблокирован.
Отправь заказ строками с размерами, например: 1000 400 2.
```

## Файлы

На первом Telegram UX этапе файлы не отправлять.

Не отправлять:

- `COREL_EXPORT.xlsx`;
- `LAST_ORDER_RESULT.txt`;
- Corel-файлы;
- архивы;
- изображения;
- PDF;
- любые производственные файлы.

Отправка файлов через Telegram — отдельный будущий слой с отдельным планом.

## Фото и Vision

Фото и Vision на первом этапе не делать.

Бот не должен:

- принимать фото как источник заказа;
- распознавать текст с фото;
- вызывать Vision;
- подключать внешние API.

## Команды

На первом минимальном этапе можно обойтись без сложного меню.

Достаточно простого поведения:

```text
любой обычный текст -> проверить как заказ
/start -> коротко объяснить, что можно отправить текст заказа
/help -> показать пример формата
```

Команды не должны запускать Excel, polling, базу, Vision, API или старую систему.

## Тон ответа

Ответ должен быть коротким и рабочим:

- без лишней технической информации;
- без цен и стоимости;
- без материалов и ЛКМ;
- с ясным следующим действием.

## Итог

Минимальный Telegram UX — это не новая производственная система, а удобный способ отправить текст заказа и получить тот же безопасный результат, который уже даёт локальная версия Malyarka Clean.
