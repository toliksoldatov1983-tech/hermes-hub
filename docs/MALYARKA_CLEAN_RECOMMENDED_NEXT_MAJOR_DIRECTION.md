Дата: 2026-06-14

Серия:

```text
Серия 064–066 — Выбор следующего крупного направления после локальной версии
BATCH_SERIES_064_066_NEXT_MAJOR_DIRECTION_SELECTION
```

## Рекомендация

Следующий крупный этап рекомендуется выбрать так:

```text
Telegram-слой
```

Причина: локальная версия уже умеет принять заказ из файла, проверить его, сохранить результат и создать Excel для Corel только для `clean`. Telegram может сделать эту рабочую цепочку удобнее для повседневного использования.

## Почему Telegram логичен следующим

- пользователь сможет отправлять текст заказа без ручной работы с файлами;
- текущий проверенный pipeline уже есть;
- Telegram может стать удобной оболочкой над готовым ядром;
- первый слой можно сделать минимальным и безопасным.

## Важное ограничение

Сейчас Telegram НЕ реализовывать.

В рамках этой серии:

```text
код Telegram не писать
Telegram token не трогать
.env не читать и не менять
polling не запускать
старый bot.py не использовать как активный
Vision не подключать
API не подключать
базу данных не трогать
цены и ЛКМ не считать
```

## Что нужно перед реализацией

Перед любой реализацией Telegram нужен отдельный безопасный план:

- что бот принимает;
- что бот отвечает;
- где хранится токен;
- как не раскрывать секреты;
- как не запускать polling случайно;
- как отделить новый Telegram-слой от старого `bot.py`;
- какие тесты и dry-run проверки нужны до live-запуска.
