# Malyarka Clean: приёмка пользовательского ввода и следующий удобный слой

Дата: 2026-06-13

Пакет:

```text
BATCH_032_USER_INPUT_ACCEPTANCE_AND_NEXT_UI_PLANNING
```

## 1. Что принято

Простой пользовательский ввод заказа считается принятым.

Работает:

- запуск через `.cmd`;
- заказ можно вставить вручную;
- завершение ввода отдельной строкой `ГОТОВО`;
- результат выводится по-русски;
- три базовых сценария проходят.

Путь для Win+R:

```text
E:\Hermes-Hub\ВВЕСТИ_ЗАКАЗ_ВРУЧНУЮ.cmd
```

## 2. Что проверено

Чистый заказ:

```text
1000 400 2
700 300
```

Ожидаемо:

- `status = clean`;
- площадь считается;
- Corel-строки готовы.

Заказ с мусором:

```text
1000 400 2
мусор
700 300
```

Ожидаемо:

- `status = has_disputes`;
- площадь считается только по нормальным строкам;
- Corel-строки заблокированы.

Полностью мусорный заказ:

```text
привет
ничего непонятно
```

Ожидаемо:

- `status = empty_or_invalid`;
- площадь `0`;
- Corel-строки заблокированы.

## 3. Что уже удобно

- Не нужно писать `PYTHONPATH`.
- Не нужно редактировать Python-код.
- Не нужен Telegram.
- Не нужен Vision.
- Не нужен API.
- Не нужна база данных.
- Не нужен Excel/Corel export.

## 4. Что пока неудобно

- Это всё ещё консольное окно.
- Результат можно посмотреть, но он не сохраняется отдельным файлом заказа.
- Нет красивого интерфейса.
- Нет кнопок.
- Нет истории заказов.

## 5. Следующие удобные варианты

Можно рассмотреть:

- сохранять результат последнего заказа в текстовый файл;
- сделать шаблон входного файла заказа;
- сделать простой локальный режим: положил заказ в `.txt` -> получил результат;
- позже рассмотреть простой локальный интерфейс.

Telegram, Vision, API, база данных и Excel/Corel export сейчас не подключаются.
