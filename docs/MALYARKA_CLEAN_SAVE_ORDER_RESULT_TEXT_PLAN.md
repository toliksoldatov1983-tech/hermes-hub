# Malyarka Clean: план сохранения результата заказа в текстовый файл

Дата: 2026-06-13

Пакет:

```text
BATCH_033_SAVE_ORDER_RESULT_TEXT_PLANNING
```

## 1. Зачем сохранять результат заказа

Сохранение результата нужно, чтобы:

- пользователь не терял результат после закрытия окна;
- результат можно было открыть обычным Блокнотом;
- было видно `status`, площадь, спорные строки и Corel-строки;
- пока не переходить к Excel/Corel export.

Это должен быть простой промежуточный слой между консольной проверкой и будущими более удобными интерфейсами.

## 2. Что должен содержать будущий текстовый результат

Будущий файл результата должен содержать:

- дату и время проверки;
- исходный текст заказа;
- `status`;
- `confirmed_rows`;
- `disputed_rows`;
- `total_area_m2`;
- `export_blocked`;
- `corel_rows`;
- `reason`, если результат заблокирован.

## 3. Куда сохранять файл

Планируемый путь:

```text
E:\Hermes-Hub\outputs\LAST_ORDER_RESULT.txt
```

В этом пакете папка `outputs` не создаётся. Её можно будет создать в следующем техническом пакете.

## 4. Правила сохранения

- Сохранять только обычный `.txt`.
- Не создавать Excel-файл.
- Не создавать Corel-файл.
- Не делать производственный export.
- Не трогать базу данных.
- Если папки `outputs` нет, её можно создать в следующем техническом пакете.
- Последний результат можно перезаписывать, чтобы не плодить мусор.

## 5. Как это должно быть удобно пользователю

Ожидаемый будущий сценарий:

1. Пользователь запускает ввод заказа.
2. Вставляет текст заказа.
3. Получает результат на экране.
4. Результат автоматически сохраняется в `LAST_ORDER_RESULT.txt`.
5. Пользователь открывает результат через Блокнот.

## 6. Что не входит

- Excel-файл;
- Corel-файл;
- экспорт на диск для производства;
- Telegram;
- Vision;
- API;
- база данных;
- цены;
- стоимость;
- ЛКМ;
- материалы;
- производство;
- CNC-обучение;
- новые агенты.
