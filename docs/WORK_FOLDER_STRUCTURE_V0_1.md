# План структуры рабочей папки заказов v0.1

Дата: 2026-06-14

Серия:

```text
Серия 077–079 — План структуры рабочей папки заказов
BATCH_SERIES_077_079_WORK_ORDERS_FOLDER_STRUCTURE_PLANNING
```

## Важная граница

Это только текстовый план будущей рабочей папки заказов.

Реальные папки на диске `E:\` в этой серии не создаются.

Нельзя без отдельного разрешения пользователя:

- создавать `E:\РАБОТА`;
- изменять старую папку `E:\Заказы 2026`;
- переносить реальные заказы;
- переименовывать реальные папки;
- менять реальные `.cdr`, `.art`, `.xlsx` файлы;
- запускать Corel, ArtCAM, CNC или Excel/Corel automation.

## Главная структура

Будущая рабочая папка:

```text
E:\РАБОТА
├─ 01_ЗАКАЗЫ
├─ 02_ШАБЛОНЫ
├─ 03_ИНСТРУМЕНТЫ
├─ 04_АРХИВ
└─ 05_РАЗОБРАТЬ
```

## Значение главных папок

`01_ЗАКАЗЫ` — текущие и новые рабочие заказы.

`02_ШАБЛОНЫ` — будущие шаблоны Excel, структуры заказа, типовые файлы.

`03_ИНСТРУМЕНТЫ` — инструкции, памятки, будущие вспомогательные материалы.

`04_АРХИВ` — завершённые или перенесённые старые заказы, только после отдельного решения пользователя.

`05_РАЗОБРАТЬ` — временное место для файлов, которые ещё не разобраны.

## Структура одного заказа

Базовая структура папки заказа v0.1:

```text
Папка_заказа
├─ 01_Исходные
├─ 02_Corel
├─ 03_ArtCAM
├─ 04_Excel
└─ 05_Покраска
```

## Назначение папок заказа

`01_Исходные` — то, что пришло от клиента: размеры, ТЗ, картинки, переписка, таблицы, если они есть отдельными файлами.

`02_Corel` — `.cdr` файлы, отрисовка, раскладка, визуальные пометки и примечания внутри Corel.

`03_ArtCAM` — `.art` файлы и подготовка под ЧПУ.

`04_Excel` — Excel-файлы заказа, списки деталей, будущий экспорт в Corel, таблицы.

`05_Покраска` — цвет, номер краски, материал, толщина, данные для малярного цеха, если заказ связан с покраской или цветом.

## Зафиксированные решения v0.1

- Отдельная папка `Фото_заметки` пока не нужна.
- Скриншоты, визуальные пометки и примечания можно хранить внутри Corel-файла.
- Сложность заказа фиксируется в карточке заказа, а не добавлением новых верхних папок.
- Будущий итоговый файл заказа пока не реализуется, но должен быть отмечен как будущий этап.
- Структура подходит для простого заказа `Абай планки`.
- Структура подходит для сложного заказа `Петр столики`.
- Структуру одного заказа принять как `v0.1`.
- Старую папку `E:\Заказы 2026` не трогать.
- Новую структуру `E:\РАБОТА` пока не создавать без отдельного разрешения пользователя.

## Будущий итоговый файл заказа

В будущем для каждого заказа можно добавить отдельный итоговый файл карточки заказа.

Пока он не реализуется.

Возможная будущая идея:

```text
Папка_заказа\ORDER_CARD.md
```

В такой карточке можно будет фиксировать:

- тип изделия;
- материал;
- толщину;
- цвет;
- номер краски;
- сложность;
- операции;
- ссылки на Corel / ArtCAM / Excel;
- статус заказа;
- заметки по производству.

## Пример 1 — Абай планки

Будущий путь:

```text
E:\РАБОТА\01_ЗАКАЗЫ\2026\06_Июнь\Абай_планки
```

Факты:

- тип изделия: детали / ЧПУ;
- операция: простая резка / раскрой;
- материал: HDF;
- толщина: 3 мм;
- цвет: RAL 9005 JET BLACK / чёрный;
- размеры:
  - 2575 x 40 — 2 шт;
  - 2547 x 30 — 2 шт.

Подходящая структура:

```text
Абай_планки
├─ 01_Исходные
├─ 02_Corel
├─ 03_ArtCAM
├─ 04_Excel
└─ 05_Покраска
```

Почему структура подходит:

- исходные размеры можно хранить в `01_Исходные`;
- `.cdr` файл и визуальные пометки — в `02_Corel`;
- подготовку под ЧПУ — в `03_ArtCAM`;
- будущий Excel / список деталей — в `04_Excel`;
- цвет, материал и толщину — в `05_Покраска` или будущей карточке заказа.

## Пример 2 — Петр столики

Будущий путь:

```text
E:\РАБОТА\01_ЗАКАЗЫ\2026\06_Июнь\Петр_столики
```

Факты:

- тип изделия: столики;
- материал: шпонированные детали;
- стиль / форма: модерн, сложный контур, закругления;
- тип обработки: сложный контурный CNC-заказ;
- есть внутренняя выборка отдельной операцией в ArtCAM;
- Corel-файл: `Безымянный-1.cdr`;
- ArtCAM-файл: `столики.art`;
- покраска / цвет / номер краски: не указано.

Подходящая структура:

```text
Петр_столики
├─ 01_Исходные
├─ 02_Corel
├─ 03_ArtCAM
├─ 04_Excel
└─ 05_Покраска
```

Почему структура подходит:

- исходные требования и заметки можно хранить в `01_Исходные`;
- `Безымянный-1.cdr`, контуры и визуальные пометки — в `02_Corel`;
- `столики.art`, контурный CNC и внутренняя выборка — в `03_ArtCAM`;
- будущие таблицы деталей — в `04_Excel`;
- если позже появится цвет или номер краски, это пойдёт в `05_Покраска`.

Сложность заказа `Петр столики` не требует новых верхних папок. Она должна быть отражена в будущей карточке заказа.

## Итог v0.1

Структура `E:\РАБОТА` и структура одного заказа принимаются как план `v0.1`.

Этот план можно использовать как основу для будущего отдельного пакета, где пользователь отдельно разрешит создать реальные папки.

До такого разрешения это только документация.
