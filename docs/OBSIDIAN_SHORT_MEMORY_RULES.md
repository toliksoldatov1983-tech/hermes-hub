# OBSIDIAN SHORT MEMORY RULES — Правила для оперативной памяти

Дата: 2026-06-21
Применение: Obsidian vault `E:\Hermes-General\obsidian-short-memory\`

---

## Назначение

Short memory = **текущий фокус** — то, что актуально прямо сейчас:
- Ближайшие задачи (на сегодня / на неделю).
- Активные блокеры.
- Текущие идеи и заметки.
- Черновики.

**Срок жизни:** дни, максимум недели. Не архив.

---

## 🟢 Что хранить в Short Memory

| Папка | Что | Пример |
|-------|-----|--------|
| `00_Inbox` | Быстрые заметки, идеи «на лету» | «Проверить цены на пластик 3мм» |
| `01_Daily` | Ежедневник: что сделать сегодня | «Доделать regression checklist» |
| `02_Ideas` | Идеи для будущего (не задачи) | «Автоматический расчёт себестоимости» |
| `03_Tasks` | **Личные задачи**, НЕ `TASK_QUEUE.md` | «Позвонить поставщику» |
| `04_Drafts` | Черновики сообщений, писем | Черновик ответа клиенту |
| `05_To_Process` | Входящие на обработку | Скриншот заказа на расшифровку |

---

## 🚫 Что НЕ хранить в Short Memory

| Не хранить | Почему | Где должно быть |
|-----------|--------|-----------------|
| **Дубликат TASK_QUEUE.md** | Расходится с Hub | Hermes Hub |
| **Дубликат WHAT_NEXT.md** | Расходится с Hub | Hermes Hub |
| **Дубликат BLOCKERS.md** | Расходится с Hub | Hermes Hub |
| **Runtime-state** | Устаревает мгновенно | Hermes Hub |
| **Секреты** | `.env`, token | **Нигде** |
| **Устаревшие задачи** | Захламляет | Удалить после выполнения |

---

## 🔄 Как обновлять после завершения batch

1. Батч завершён в Hub → `TASK_QUEUE.md` обновлён.
2. Открыть Short Memory `03_Tasks`.
3. Найти задачу, соответствующую батчу.
4. Пометить как выполненную или удалить.
5. Если в `00_Inbox` есть связанные заметки → обработать.

---

## 🧹 Как очищать (еженедельно)

| Папка | Действие |
|-------|----------|
| `00_Inbox` | Разобрать: в задачи / в идеи / удалить |
| `01_Daily` | Прошедшие дни → в `10_Sessions` Long Memory (итоги) |
| `02_Ideas` | Актуальные оставить, остальные → архив |
| `03_Tasks` | Выполненные удалить, неактуальные удалить |
| `04_Drafts` | Отправленные удалить, остальные — review |
| `05_To_Process` | Обработанные удалить |

---

## 🔍 Еженедельная сверка с Hub

1. Открыть `TASK_QUEUE.md` (Hub).
2. Открыть Short Memory `03_Tasks`.
3. Задачи из Short Memory, которых нет в Hub → либо добавить в Hub, либо это личные задачи.
4. Задачи из Hub, которые есть в Short Memory → статусы должны совпадать.

**Правило:** если задача есть в Hub — она главная. Short Memory — копия для фокуса, не источник правды.

---

## ⚡ Правило срока жизни

| Возраст | Действие |
|---------|----------|
| 0-3 дня | Активно |
| 3-7 дней | На проверку: актуально? |
| 7-14 дней | В архив Long Memory или удалить |
| 14+ дней | Удалить (устарело) |

---

## 📋 Памятка

```
ФОКУС → Short Memory (текущие задачи, идеи, черновики)
STATE → Hermes Hub (TASK_QUEUE.md, BLOCKERS.md)
НЕ ДУБЛИРОВАТЬ → Hub-задачи как единственный список
ОЧИЩАТЬ → еженедельно
```
