# План будущего использования архива старой Малярки

BATCH_SERIES_080_082_OLD_MALYARKA_ARCHIVE_FUTURE_PLAN

## Важная граница

Это только текстовый план.
Архив старой Малярки полезен как источник идей, но он не является актуальными правилами Hermes.
Ничего из архива не переносится автоматически.
Каждую группу данных нужно позже принимать отдельно, отдельным планом и отдельным решением пользователя.
Цены, нормы, зарплаты, материалы и склад считаются неподтвержденными до отдельного решения пользователя.
Старые токены, JSON, Telegram-код, webhook, Google Apps Script и старый код нельзя использовать как активную систему.

## Общий статус архива

Архив старой Малярки имеет статус:

```text
archive-only / source of future ideas
```

Это значит:

- можно использовать архив как источник мыслей, терминов, примеров и кандидатов;
- нельзя считать архивные данные действующими правилами;
- нельзя переносить старую логику без проверки;
- нельзя принимать архивные цены как актуальные;
- нельзя запускать старый Telegram;
- нельзя использовать старые токены, ключи, webhook или JSON как действующие.

Рекомендуемая последовательность:

1. Сначала завершить безопасный Telegram-слой.
2. Потом отдельно разобрать архивные шаблоны.
3. Потом отдельно разобрать цены.
4. Потом отдельно разобрать экономику заказа.
5. Потом отдельно разобрать зарплаты.
6. Потом отдельно разобрать склад.
7. Потом спроектировать итоговый файл заказа.
8. Потом только после подтверждения пользователя переносить отдельные решения в Hermes.

## Что не делается сейчас

В этой серии не делается:

- перенос архивных правил в актуальные;
- принятие архивных цен как действующих;
- использование старых токенов;
- запуск Telegram;
- запуск polling;
- чтение `.env`;
- изменение кода Hermes / Malyarka Clean;
- создание новых автоматизаций;

- подключение API;
- работа с базой данных;
