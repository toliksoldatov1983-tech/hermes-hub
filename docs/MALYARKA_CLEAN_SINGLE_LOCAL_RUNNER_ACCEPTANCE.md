## Что создано

Единый локальный запуск заказа создан.

Главный запускатель:

```text
E:\Hermes-Hub\ЗАПУСТИТЬ_ЗАКАЗ.cmd
```

Runner:

```text
E:\Hermes-Hub\projects\malyarka-clean\tools\run_local_order.py
```

## Что делает единый запуск

Он:

- читает `E:\Hermes-Hub\inputs\ORDER_INPUT.txt`;
- проверяет заказ через уже существующее ядро;
- всегда сохраняет `E:\Hermes-Hub\outputs\LAST_ORDER_RESULT.txt`;
- если заказ чистый — создаёт или обновляет `E:\Hermes-Hub\outputs\COREL_EXPORT.xlsx`;
- если заказ спорный или пустой — не создаёт новый Excel и показывает причину блокировки.

## Принятые сценарии

```text
clean -> текстовый результат сохранён, COREL_EXPORT.xlsx создан
has_disputes -> текстовый результат сохранён, Excel не создаётся/не обновляется
empty_or_invalid -> текстовый результат сохранён, Excel не создаётся/не обновляется
```

## Важно

Это локальный режим.

Это не:

- Telegram;
- Vision;
- API;
- база данных;
- расчёт цены;
- производство;
- Corel-файл.

## Что не менялось

Не менялись:

- парсер;
- расчёт площади;
- правила спорных строк.
