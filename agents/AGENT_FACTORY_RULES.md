# Agent Factory Rules

Date: 2026-06-17
Project: Hermes Hub
Status: proposed (markdown-only spec)

## 1. Что такое Agent Factory

**Agent Factory** — это система управления суб-агентами в Hermes Hub.

Она позволяет:
- Создавать новых агентов по спецификации (AGENT_SPEC)
- Регистрировать агентов в реестре (AGENT_REGISTRY)
- Управлять жизненным циклом агентов
- Определять правила безопасности для каждого агента

**Не код. Не runtime. Не API.** Это проектная документация и правила.

## 2. Как создавать суб-агентов

Суб-агент создаётся **только через AGENT_SPEC** — markdown-документ, который описывает:

| Раздел | Содержание |
|--------|-----------|
| **AGENT_SPEC.md** | Полная спецификация: роль, цель, статус, правила |
| **QUESTIONS.md** | Список вопросов, которые агент задаёт |
| **INTAKE_CARD_TEMPLATE.md** | Шаблон карточки (для intake-агентов) |
| **RESPONSE_TEMPLATES.md** | Шаблоны ответов |
| **SAFETY_RULES.md** | Что агенту запрещено |
| **HANDOFF_TO_*.md** | Правила передачи другому агенту |

## 3. Статусы агентов

| Статус | Значение |
|--------|---------|
| `proposed` | Спецификация создана, ждёт утверждения |
| `accepted` | Спецификация утверждена пользователем |
| `active` | Агент работает (в production или тестовом режиме) |
| `blocked` | Агент приостановлен (блокер, ошибка, safety risk) |
| `archived` | Агент выведен из эксплуатации, спецификация сохранена |

## 4. Цепочка агентов (как они взаимодействуют)

```text
Клиент → Sales + Client Intake Agent → Malyarka Agent → Corel Export Agent
                ↓ (рукопожатие)              ↓ (расчёт)         ↓ (файлы)
          Intake Card                  Order Result        Corel .cdr
```

Агенты не общаются друг с другом напрямую. Они передают **структурированные объекты** (Intake Card, Order Result) через Hermes Hub.

## 5. Правила фабрики

1. Агент не может быть создан без AGENT_SPEC.
2. Агент не может перейти в `active` без утверждения пользователем.
3. Каждый агент имеет свои SAFETY_RULES — они не могут быть слабее глобальных SAFETY_RULES.
4. Агент не может читать secrets, token, .env, config.py.
5. Агент не может подключаться к серверу без отдельного разрешения.
6. Агент не может отправлять сообщения в Telegram без live-разрешения.
7. Агент не может обрабатывать реальные заказы без разрешения.
8. Все агенты — markdown-first. Код пишется только после accepted-статуса.

## 6. Именование агентов

```text
agents/<agent_name>/
  AGENT_SPEC.md
  QUESTIONS.md
  INTAKE_CARD_TEMPLATE.md
  RESPONSE_TEMPLATES.md
  SAFETY_RULES.md
  HANDOFF_TO_<target_agent>.md
```

## 7. Глобальные запреты для всех агентов

- ❌ Выдумывать цены
- ❌ Обещать сроки
- ❌ Принимать заказ в производство
- ❌ Трогать сервер
- ❌ Читать secrets
- ❌ Работать с реальными заказами без разрешения
- ❌ Писать клиенту автоматически без live-разрешения
- ❌ Делать commit/push
