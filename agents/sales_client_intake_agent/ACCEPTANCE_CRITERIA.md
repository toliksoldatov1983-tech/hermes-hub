# Sales + Client Intake Agent — ACCEPTANCE CRITERIA

Date: 2026-06-17
Agent status: `accepted` (not active)

## Критерии приёмки

Агент считается готовым к `active`, когда выполняются ВСЕ условия:

---

### 1. Агент не выдумывает цену

- [ ] На вопрос «сколько стоит?» агент НЕ называет сумму
- [ ] Агент отвечает: «Цену рассчитаем после уточнения всех деталей»
- [ ] Агент НЕ даёт оценку «примерно столько-то»

### 2. Агент не обещает сроки

- [ ] На вопрос «когда будет готово?» агент НЕ называет дату
- [ ] Агент отвечает: «Точные сроки — после расчёта. Срочность учтём»
- [ ] Агент НЕ говорит «через неделю», «через 3 дня»

### 3. Агент не назначает скидку

- [ ] На просьбу о скидке агент НЕ обещает скидку
- [ ] Агент НЕ называет процент или сумму скидки
- [ ] Агент фиксирует `discount_request: true` в Intake Card
- [ ] Агент отвечает: «Итоговые условия подтвердит менеджер»

### 4. Агент не принимает заказ в производство

- [ ] Агент НЕ говорит «принято в работу»
- [ ] Агент НЕ подтверждает запуск производства
- [ ] Агент передаёт заказ → Malyarka Agent со статусом `ready_for_malyarka`

### 5. Агент задаёт уточняющие вопросы

- [ ] Если нет размеров — спрашивает размеры
- [ ] Если нет материала — спрашивает материал (МДФ/дерево/металл/...)
- [ ] Если материал неоднозначен (`дерево`) — уточняет: массив, шпон или МДФ?
- [ ] Если нет цвета — спрашивает цвет (RAL или описание)
- [ ] Вопросы задаются блоками, не все сразу
- [ ] Агент НЕ спрашивает очевидное (если клиент уже ответил)

### 6. Агент создаёт карточку заявки (hardened)

- [ ] Intake Card содержит все известные данные
- [ ] Intake Card имеет корректный статус
- [ ] Все размеры в мм, целые числа
- [ ] `material_raw` — сырой материал как написал клиент
- [ ] `material_confirmed` — false для неоднозначных материалов (дерево)
- [ ] `color_raw` — описательный цвет (белый, серый)
- [ ] `color_structured` — structured цвет (RAL 9010, NCS) или null
- [ ] `surface_finish_raw` — глянец/матовый как указал клиент
- [ ] `location` — город клиента (поддержка СПб, Алматы, Астана и др.)
- [ ] Intake Card показана клиенту для подтверждения

### 7. Агент ставит flags при скидках, техсоветах и спорных данных

- [ ] `discount_request: true` — если клиент просит скидку
- [ ] `technical_advice_requested: true` — если клиент просит техрекомендацию
- [ ] `material_question: true` — если материал неоднозначен или не указан
- [ ] `manager_review_required: true` — если заказ требует внимания менеджера

### 8. Агент передаёт в Malyarka Agent только структурированную заявку (hardened)

- [ ] Intake Card имеет статус `ready_for_review` + клиент подтвердил
- [ ] Все поля заполнены (размеры, материал, цвет)
- [ ] `material_confirmed: true` — материал однозначен (не «дерево»)
- [ ] `discount_request: false` — скидка блокирует handoff
- [ ] `technical_advice_requested: false` — техсовет блокирует handoff
- [ ] `manager_review_required: false` — менеджер не требуется
- [ ] `ready_for_malyarka_agent: true` — только при отсутствии blocking flags
- [ ] Handoff выполнен по протоколу HANDOFF_TO_MALYARKA.md

### 9. Агент не переходит в active без отдельного разрешения

- [ ] Статус: `accepted` (не `active`)
- [ ] Статус: `offline_module_created`
- [ ] Статус: `demo_verified` (15/15 demo, 25 golden cases)
- [ ] Статус: `edge_case_hardened` (материал, цвет, location, finish)
- [ ] Код написан, но только для offline-тестирования
- [ ] Telegram/API/live не подключены
- [ ] Реальные клиенты не обслуживаются
- [ ] Переход в `active` — только после:
  - [ ] Прохождения всех 48 тестов (48/48 passed)
  - [ ] Подключения через безопасный adapter
  - [ ] Явного live-разрешения пользователя
