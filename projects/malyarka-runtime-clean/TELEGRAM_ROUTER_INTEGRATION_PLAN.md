# TELEGRAM_ROUTER_INTEGRATION_PLAN

state_version: 0.8.4

## 1. Цель

Безопасно подключить router/state machine к Telegram-слою так, чтобы:

- команды режимов работали в Telegram;
- текущий preview текстовых заказов не сломался;
- `/чат` не отправлял текст в parser заказа;
- `neutral` не разбирал похожий на заказ текст автоматически;
- только `order` mode отправлял текст в parser заказа;
- DeepSeek API не вызывался;
- ключи не читались;
- polling не запускался на этапе разработки.

## 2. Текущая ситуация

Уже есть:

- `malyarka_telegram/modes.py` — режимы и команды;
- `malyarka_telegram/session.py` — in-memory `user_id -> current_mode`;
- `malyarka_telegram/router.py` — routing logic;
- тесты `modes/router`;
- `app.py` и `handlers.py` пока не подключены к router;
- рабочий Telegram preview сейчас работает старым путём.

## 3. Будущий безопасный кодовый пакет подключения

Разрешённые будущие файлы:

- `malyarka_telegram/handlers.py`
- `malyarka_telegram/app.py`, только если реально нужно передать routing result в Telegram
- `tests/test_malyarka_telegram_router_integration.py`
- `tests/test_malyarka_telegram_safe_launch.py`
- `tests/test_malyarka_telegram_messages.py`, если нужно проверить preview

Файлы и данные, которые нельзя трогать:

- `bot.py`
- `.env`
- `orders.db`
- `.venv`
- `.git`
- реальные фото
- токены
- ключи
- личные данные

## 4. Минимальная схема подключения

Предпочтительный вариант:

- `handle_text_message(text, user_id, session_store)` или аналогичная безопасная обёртка вызывает router;
- если router возвращает `should_parse_order=True`, использовать текущий local order preview;
- если `should_parse_order=False`, возвращать сообщение режима;
- copy-кнопки и старый preview для `order` mode должны остаться рабочими.

Совместимость:

- Если текущий `handle_text_message` сейчас не принимает `user_id/session_store`, сохранить старую сигнатуру для существующих тестов.
- Добавить новую функцию или обёртку для режимов.
- Не ломать текущие вызовы.

## 5. Поведение команд

- `/заказ` включает `order` mode и отвечает: `Включён режим заказа. Пришлите размеры.`
- `/чат` включает `chat` mode и отвечает: `Включён режим чата. Можно обсуждать идеи и вопросы.`
- `/инженер` включает `engineer` mode и отвечает: `Включён инженерный режим. Можно готовить задачу или выжимку.`
- `/выжимка` включает `summary` mode и отвечает: `Включён режим выжимки. Пришлите текст идеи или разговора.`
- `/правила` включает `rules` mode и отвечает: `Включён режим правил. Напишите, какое правило проверить.`

## 6. Сохранение старого поведения

Существующие сценарии должны остаться:

- чистый текстовый заказ в `order` mode даёт preview;
- `1000*400` распознаётся;
- кнопка `Скопировать для Corel` остаётся;
- спорная строка блокирует экспорт;
- photo handler остаётся заглушкой;
- `app --check` не запускает polling.

## 7. Тесты будущего кодового этапа

- команда `/заказ` включает `order` mode;
- после `/заказ` текст `1000*400` идёт в parser заказа;
- команда `/чат` включает `chat` mode;
- в `/чат` текст `1000*400` не идёт в parser заказа;
- `neutral` mode не разбирает похожий на заказ текст автоматически;
- `/инженер` не меняет код;
- `/выжимка` не создаёт задачу автоматически;
- `/правила` не меняет правила напрямую;
- старый `handle_text_message` без `user_id` продолжает работать, если он есть;
- copy-кнопки для order preview не ломаются;
- `app.py` import не запускает polling;
- `malyarka_telegram.app --check` показывает `polling_started=false`;
- DeepSeek API не вызывается;
- ключи не читаются;
- `orders.db` не используется.

## 8. Риски и защита

Риск: сломать текущий preview.

Защита:

- тесты до подключения;
- сохранить текущий local order preview;
- подключать router через совместимую обёртку.

Риск: отправить разговорный текст в parser заказа.

Защита:

- `should_parse_order=True` только в `order` mode;
- в `/чат` и `neutral` mode order-like текст только предлагает перейти в `/заказ`.

Риск: изменить сигнатуру функции и сломать тесты.

Защита:

- сохранить старую сигнатуру `handle_text_message`, если она используется;
- добавить новую совместимую функцию-обёртку для режимов.

Риск: случайно подключить AI/API.

Защита:

- `should_call_ai=False` во всех режимах на этом этапе;
- не импортировать DeepSeek/API SDK в Telegram router integration;
- покрыть тестом отсутствие API-вызова.

Риск: тронуть `orders.db`.

Защита:

- session хранить только in-memory;
- не менять схему базы;
- покрыть тестом отсутствие использования `orders.db`.

## 9. Что не делать сейчас

- не писать код подключения;
- не менять `handlers.py`;
- не менять `app.py`;
- не запускать polling;
- не вызывать DeepSeek API;
- не читать ключи;
- не трогать `orders.db`;
- не трогать `bot.py`;
- не делать `git add`, commit, push.
